numpy关于copy有三种情况，完全不复制、浅复制（shadow copy）和深复制（deep copy）。
而 b = a[:] 这种形式就属于第二种，即视图，这本质上是一种切片操作（slicing），所有的切片操作返回的都是视图。具体来说，b = a[:]会创建一个新的对象 b（所以 id(b) 和id(a) 返回的结果是不一样的），但是 b 的数据完全来自于a，和 a 保持完全一致，换句话说，b的数据完全由a保管，他们两个的数据变化是一致的。

b = a 和 b = a[:] 的差别就在于后者会创建新的对象，前者不会。两种方式都会导致 a 和 b 的数据相互影响。

集成学习算法

集成学习的主要思想是利用一定的手段学习出多个分类器，而且这多个分类器要求是弱分类器，然后将多个分类器进行组合公共预测。核心思想就是如何训练处多个弱分类器以及如何将这些弱分类器进行组合。

基于Bagging

bagging方法bootstrap aggregating的缩写，采用的是随机有放回的选择训练数据然后构造分类器，最后组合。
代表算法：随机森林

Random Forest

基本思想：构造很多棵决策树，形成一个森林，然后用这些决策树共同决策输出类别是什么。
随机森林算法的过程中，有两个随机过程：第一个就是输入数据是随机地从整体的训练数据中选取一部分作为一棵决策树的构建，而且是有放回的选取；第二个就是每棵决策树的构建所需要特征是从整体的特征集随机选取的，这两个随机过程使得随机森林很大程度上避免了过拟合现象的出现。

Boosting

代表算法：AdaBoost、GBDT、XGBOOST

AdaBoosting

AdaBoost算法是基于Boosting思想的机器学习算法，其中AdaBoost是Adaptive Boosting的缩写，AdaBoost是一种迭代型的算法，其核心思想是针对同一个训练集训练不同的学习算法，即弱学习算法，然后将这些弱学习算法集合起来，构造一个更强的最终学习算法。
为了构造出一个强的学习算法，首先需要选定一个弱学习算法，并利用同一个训练集不断训练弱学习算法，以提升弱学习算法的性能。在AdaBoost算法中，有两个权重，第一个数训练集中每个样本有一个权重，称为样本权重，用向量表示$D_1$；另一个是每一个弱学习算法具有一个权重，用向量表示$\alpha_1$。假设有$n$个样本的训练集，初始时，设定每个样本的权重是相等的，即，利用第一个弱学习算法对其进行学习，学习完成后进行错误率的统计：
$\varepsilon_1=\frac{error}{all}$
其中error表示被错误分类的样本数量，all表示你所有样本数量。这样就可以利用错误率计算弱学习算法的权重：
$\alpha_1=\frac{1}{2}ln(\frac{1-\varepsilon_1}{\varepsilon_1})$
在第一次学习完成后，需要重新调整样本的权重，提高在第一次分类中被错误分类的样本的权重，使得在接下来的学习中可以重点对其进行学习：

这样进行下一次的学习，等学习$t$轮后，得到了t个弱学习算法${h_1,…,h_t}$及其权重${\alpha_1,…,\alpha_t}$。

决策树

介绍

决策树又叫做decision tree，这个是一种比较简单但是又得到广泛应用的分类器的一种形式。我们一般都是通过训练的数据来搭建起决策树的模型。通过这个模型，我们可以高效的对于未知的数据进行归纳分类。

分治思想：

案例:预测小明今天出不出去打球

在上图中，如何判断选择哪个条件去区分是数据呢?
首先需要引入一个判断纯度的标准：信息熵

信息熵

H(S) = - P(是)logP(是) - P(否)logP(否)
这个标准需要是symmetric的，也就是说 4是/0否 = 0是/4否
更通用的表达式：$H(X)=-\sum_{i=1}^{n}p(x_i)logp(x_i)$
其中$P(x_{i}) $代表随机事件X为$x_{i}$的概率.

信息增益

信息增益是特征选择的一个重要指标。这个值越大，说明该特征越重要，相应的信息增益也就越大。
前面我们说过，信息熵代表随机变量的复杂度，条件熵代表在某个条件下，随机变量的复杂度。
也就是信息增益代表了在一个条件下，信息复杂度减少程度。
所以，如果选择一个特征后，信息增益最大，我们可以选择这个特征进行划分。
eg:

可以求得随机变量X（嫁与不嫁）的信息熵为：
嫁的个数为6个，占1/2，那么信息熵为-1/2log1/2-1/2log1/2 = -log1/2=0.301
现在假如我知道了一个男生的身高信息。身高有三个可能的取值{矮，中，高}矮包括{1,2,3,5,6,11,12}，嫁的个数为1个，不嫁的个数为6个中包括{8,9} ，嫁的个数为2个，不嫁的个数为0个高包括{4,7,10}，嫁的个数为3个，不嫁的个数为0。
我们先求出公式对应的:
H(Y|X = 矮) = -1/7log1/7-6/7log6/7=0.178
H(Y|X=中) = -1log1-0 = 0
H(Y|X=高） = -1log1-0=0
p(X = 矮) = 7/12
p(X =中) = 2/12
p(X=高) = 3/12
则可以得出条件熵为：7/120.178+2/120+3/12*0 = 0.103
那么我们知道信息熵与条件熵相减就是我们的信息增益，为0.301-0.103=0.198
所以我们可以得出我们在知道了身高这个信息之后，信息增益是0.198

基尼系数

CART算法采用基尼系数来选择划分属性。数据集D的纯度用基尼系数来度量：
$Gini(D) = \sum_{k=1}^{|y|}\sum_{k^,\neq k}^{}p_k^2=1-\sum_{k=1}^{|y|}p_k^2$
直观上说，Gini反应了从数据集D中随机抽取两个样本，其类别标注不一致的概率。因此，基尼系数越小，数据集D的纯度越高。
所以，我们在候选属性集合A中，选择哪个使得划分后基尼指数最小的属性作为最优划分属性。

包括ID3算法、C4.5算法、CART算法

ID3算法

  1. 对当前样本集合，计算所有属性的信息增益
  2. 选择信息增益最大的属性作为测试属性
  3. 把测试属性取值相同的样本划分为同一个子样本集
  4. 如果子样本集的类别值唯一，则分支为叶子节点；否则对子样本集递归调用本算法。

如何在hexo中渲染MathJax数学公式

Posted on 2018-05-16 | In 博客技巧

在用markdown写技术文档时，免不了会碰到数学公式。常用的Markdown编辑器都会集成Mathjax，用来渲染文档中的类Latex格式书写的数学公式。基于Hexo搭建的个人博客，默认情况下渲染数学公式却会出现各种各样的问题。

原因

Hexo默认使用”hexo-renderer-marked”引擎渲染网页，该引擎会把一些特殊的markdown符号转换为相应的html标签，比如在markdown语法中，下划线’_’代表斜体，会被渲染引擎处理为标签。

因为类Latex格式书写的数学公式下划线 ‘_’ 表示下标，有特殊的含义，如果被强制转换为标签，那么MathJax引擎在渲染数学公式的时候就会出错。例如，$x_i$在开始被渲染的时候，处理为$xi$，这样MathJax引擎就认为该公式有语法错误，因为不会渲染。

类似的语义冲突的符号还包括’*’, ‘{‘, ‘}’, ‘\’等。

解决方法

解决方案有很多，可以网上搜下，为了节省大家的时间，这里只提供亲身测试过的最靠谱的方法。

更换Hexo的markdown渲染引擎，hexo-renderer-kramed引擎是在默认的渲染引擎hexo-renderer-marked的基础上修改了一些bug，两者比较接近，也比较轻量级。

测试/预测阶段

对于给定的x，预测其输出 $\hat{y}=f(x)$

如何表示f

$f(x)=\theta_{0}x_{0}+\theta_{1}x_{1}+\theta_{2}x_{2}+…+\theta_{n}x_{n}$

过拟合与正则化

过拟合

过拟合问题:如果我们有非常多特征/模型很复杂, 我们的假设函数曲线可以对 原始数据拟合得非常好($J(\theta_0,\theta_1,…,\theta_n)=\frac{1}{2m}\sum_{i=1}^{m}(h_\theta(x^{(i)})-y^{(i)})^2\thickapprox0$), 但丧失了一般性，从而导 致对新给的待预测样本，预测效果差。
所有的模型都可能存在过拟合的风险:

  1. 更多的参数，更复杂的模型，意味着有更强的能力，但也更可能无法无天
  2. 眼见不一定为实，你看到的内容不一定是全部真实的数据分布，死记硬背不太好

正则化

  1. 控制参数幅度，不让模型“无法无天”
  2. 限制参数搜索空间
    $J(\theta)=\frac{1}{2m}[\sum_{i=1}^{m}(h_\theta(x^{(i)})-y^{(i)})^2+\lambda\sum_{j=1}^{n}\theta_j^2]$
